## anthropics/financial-services

This repository provides a collection of AI agents and plugins that execute specialized financial workflows. It facilitates the automation of complex financial tasks, such as earnings analysis, financial modeling, and general ledger reconciliation. The system integrates these capabilities with external financial data providers and the Microsoft 365 ecosystem.

The system utilizes named AI agents to encapsulate end-to-end processing for specific financial tasks. These agents are powered by plugins, which provide granular functionalities and domain-specific skills. Agents can be deployed either as Claude Managed Agent templates or as [`Cowork`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L50) plugins, offering flexibility in integration and operation.

Plugins enable integration with financial data providers such as LSEG and S&P Global, supporting AI-assisted research and report generation, as discussed in [Partner-Built Plugins for Data Integration](https://codewiki.google/github.com/anthropics/financial-services#financial-services-ai-agents-and-plugins-partner-built-plugins-for-data-integration). The system also offers specialized vertical plugins, which provide targeted automation for areas like Equity Research, Financial Analysis, Fund Administration, and Investment Banking, as outlined in [Financial Services Vertical-Specific Plugins](https://codewiki.google/github.com/anthropics/financial-services#financial-services-vertical-specific-plugins). These diverse plugins orchestrate end-to-end financial tasks.

Pre-built Managed Agent templates define agent capabilities and configurations for various financial use cases, acting as blueprints for deployments, as described in [Financial Services Managed Agent Cookbooks](https://codewiki.google/github.com/anthropics/financial-services#financial-services-managed-agent-cookbooks). These agents employ a three-tier security isolation model to manage untrusted documents and orchestrate inter-agent communication via [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) mechanisms. Agent deployment and configuration are standardized to ensure consistent operations.

The system integrates the Claude Office add-in with custom large language model backends for Microsoft 365 applications, detailed further in [Claude for Microsoft 365 Integration](https://codewiki.google/github.com/anthropics/financial-services#claude-for-microsoft-365-integration). This integration includes administrative tooling for cloud resource provisioning, manifest generation, and dynamic user configuration. Users are authenticated via Entra ID tokens, and Role-Based Access Control (RBAC) rules enforce secure access within the Microsoft 365 ecosystem.

Supporting scripts automate the deployment and management of these financial services agents, as outlined in [Agent Deployment and Orchestration Scripts](https://codewiki.google/github.com/anthropics/financial-services#agent-deployment-and-orchestration-scripts). These scripts validate configurations, orchestrate agent handoffs, and synchronize bundled skills across the platform. This automation ensures operational efficiency and consistency.

## Financial Services AI Agents and Plugins

The financial services offering provides a framework for AI agents and plugins designed to automate and enhance complex financial workflows. These agents are equipped with domain-specific skills and integrate with various data sources, operating through dual deployment mechanisms: as [`Claude Managed Agent`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L1) templates or as [`Cowork`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L50) plugins. The core components are located in the [`(root)`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/) directory.

The system is built around named agents that encapsulate end-to-end processing for specific financial tasks. These agents, defined in [`plugins/agent-plugins`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins), leverage specialized skills and data connectors to perform their functions. For instance, an agent might handle earnings reviews, general ledger reconciliation, or KYC screening by orchestrating a series of internal skills and external data calls.

The dual deployment mechanism allows for flexibility in how these AI capabilities are utilized. Managed agents are pre-built, specialized AI agents provided as templates that define their capabilities and configurations for various financial use cases. These are deployed via the [`Claude Managed Agents API`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L15) using configurations outlined in [Financial Services Managed Agent Cookbooks](https://codewiki.google/github.com/anthropics/financial-services#financial-services-managed-agent-cookbooks). Conversely, [`Cowork`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L50) plugins allow for more direct integration within existing workflows, providing specific functionalities on demand.

Plugins within the system are categorized to address different needs. [Core Financial Services Agent Plugins](https://codewiki.google/github.com/anthropics/financial-services#financial-services-ai-agents-and-plugins-core-financial-services-agent-plugins) include agents for tasks such as earnings review and financial model building. [Partner-Built Plugins for Data Integration](https://codewiki.google/github.com/anthropics/financial-services#financial-services-ai-agents-and-plugins-partner-built-plugins-for-data-integration), found in [`plugins/partner-built`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/partner-built), integrate with major financial data providers like LSEG and S&P Global, enabling AI-assisted research and the generation of formatted financial reports. Additionally, [Financial Services Vertical-Specific Plugins](https://codewiki.google/github.com/anthropics/financial-services#financial-services-vertical-specific-plugins) offer domain-specific functionalities for areas like Equity Research, Financial Analysis, and Investment Banking, providing targeted automation for specialized financial tasks.

The underlying structure of these agents and plugins ensures consistency and maintainability. Canonical system prompts for agents are defined in files like [`agents/<slug>.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L157), and their skills are typically sourced from the [`plugins/vertical-plugins`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L94) directory, then bundled into individual agents. This modular approach, along with utility scripts in [`scripts`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts) for tasks like skill synchronization and agent deployment, helps in managing and propagating changes across the system. The entire ecosystem also supports integration with Microsoft 365, allowing agents and skills to operate within applications like Excel and PowerPoint. See [Claude for Microsoft 365 Integration](https://codewiki.google/github.com/anthropics/financial-services#claude-for-microsoft-365-integration) for more details.

### Agent Workflows and Specializations

The agents are specialized for distinct financial operations:

- **Earnings Reviewer**: This agent processes earnings events for individual companies or across coverage lists. It acts as a senior equity research associate, capable of reviewing transcripts and regulatory filings, updating financial models, and drafting post-earnings notes. For example, the [`earnings-reviewer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L26) agent in [`plugins/agent-plugins/earnings-reviewer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/earnings-reviewer) utilizes skills like [`earnings-analysis`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L205) to extract guidance from transcripts, [`model-update`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L208) to revise financial models, and [`morning-note`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L209) to generate summary reports.
- **GL Reconciler**: Focused on financial accounting, the [`gl-reconciler`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L29) agent in [`plugins/agent-plugins/gl-reconciler`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/gl-reconciler) performs general ledger to subledger reconciliation. It identifies variances, traces their root causes, and generates exception reports for review, without performing ledger adjustments directly.
- **KYC Screener**: The [`kyc-screener`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L32) agent, defined in [`plugins/agent-plugins/kyc-screener`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/kyc-screener), automates Know Your Customer (KYC) screening. It parses onboarding documents to extract entity information, applies firm-specific KYC/AML rules, screens against sanctions and Politically Exposed Persons (PEP) lists, and generates escalation packets for compliance review.
- **Market Researcher**: This agent generates sector and thematic market research. The [`market-researcher`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L25) agent in [`plugins/agent-plugins/market-researcher`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/market-researcher) drafts industry overviews, competitive landscapes, peer trading comparables, and investment idea shortlists. These are compiled into structured research notes, with an option for slide pack generation.
- **Meeting Prep Agent**: Designed to assist financial advisors, the [`meeting-prep-agent`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L24) in [`plugins/agent-plugins/meeting-prep-agent`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/meeting-prep-agent) prepares briefing packs and talking points for client meetings. It gathers client relationship history, holdings, recent activity, and market context to create comprehensive preparation materials.
- **Model Builder**: This agent specializes in constructing institutional-quality financial valuation models in Excel from scratch. The [`model-builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L27) agent in [`plugins/agent-plugins/model-builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/model-builder) can build Discounted Cash Flow (DCF), Leveraged Buyout (LBO), Three-statement, and Comparables models, ensuring they are formula-driven and professionally formatted.
- **Month-End Closer**: Automating aspects of the month-end closing process, the [`month-end-closer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L30) agent in [`plugins/agent-plugins/month-end-closer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/month-end-closer) manages close checklists, generates accrual schedules, roll-forward schedules, and variance commentary, assembling a comprehensive close package.
- **Pitch Agent**: For investment banking, the [`pitch-agent`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L23) in [`plugins/agent-plugins/pitch-agent`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/pitch-agent) automates the initial drafting of pitches. It generates Excel valuation workbooks and PowerPoint pitch decks based on target companies and strategic situations.
- **Statement Auditor**: The [`statement-auditor`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L31) agent in [`plugins/agent-plugins/statement-auditor`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/statement-auditor) audits Limited Partner (LP) capital-account statements. It compares statements against fund Net Asset Value (NAV) packs, identifies discrepancies, and provides recommendations.
- **Valuation Reviewer**: This agent reviews portfolio-company valuations. The [`valuation-reviewer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L28) agent in [`plugins/agent-plugins/valuation-reviewer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/valuation-reviewer) processes General Partner (GP) valuation packages, computes fund-level NAV, and stages LP reporting.

### Partner-Built Plugins for Data Integration

The financial services offering includes plugins developed to integrate with major financial data providers, such as LSEG and S&P Global. These plugins facilitate AI-assisted research and analysis by orchestrating various data tools and generating formatted financial reports.

The LSEG plugin integrates with the LSEG Financial Analytics MCP Server. It provides access to a comprehensive suite of financial analytics tools, exposing eight high-level workflows as commands for financial analysis tasks. Each command orchestrates multiple LSEG MCP tools to perform in-depth analyses. For instance, commands exist for analyzing bond futures basis, bond relative value, FX carry trades, and option volatility. It also supports equity research, fixed income portfolio reviews, macro and rates dashboard construction, and interest rate swap curve analysis. Each command is supported by a specialized skill, such as [`plugins/partner-built/lseg/skills/bond-futures-basis`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/partner-built/lseg/skills/bond-futures-basis) or [`plugins/partner-built/lseg/skills/equity-research`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/partner-built/lseg/skills/equity-research), which embeds domain knowledge into the analysis. For more details on the available tools, refer to [`plugins/partner-built/lseg/CONNECTORS.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/partner-built/lseg/CONNECTORS.md).

The S&P Global plugin integrates S&P Global financial data and analytics into AI workflows via pre-built skills. These skills enable AI-assisted research, analysis, and document generation for financial professionals. The core functionality centers on generating specific financial outputs. For example, the skill in [`plugins/partner-built/spglobal/skills/earnings-preview-beta`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/partner-built/spglobal/skills/earnings-preview-beta) generates detailed single-company earnings preview reports as HTML files. The skill located at [`plugins/partner-built/spglobal/skills/funding-digest`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/partner-built/spglobal/skills/funding-digest) summarizes recent funding rounds into a single-page PowerPoint slide. Deployment of these managed agents is typically automated via a script such as [`../scripts/deploy-managed-agent.sh`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L5), which handles the uploading of skills, creation of leaf workers, and configuration of agents through the [`POST /v1/agents`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) API. Each agent's manifest ([`agent.yaml`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L34)) resolves to specific API fields, where system prompts can be inlined from files or directly provided as text, and skills can be referenced from plugin directories or specific paths.

A key architectural feature across many managed agents is a three-tier security isolation model, particularly for handling untrusted documents. This model typically involves:

1. A [`reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/validate.py#L8) tier: Interacts with untrusted documents, restricted to read-only tools and no external connectors, outputting length-capped, schema-validated JSON.
2. An [`Orchestrator`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/kyc-screener/README.md?plain=1#L26) tier: Does not directly handle untrusted documents but orchestrates the workflow, often connecting to read-only external services.
3. A [`Write-holder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/pitch-agent/README.md?plain=1#L27) tier: Avoids direct interaction with untrusted documents, focusing on writing outputs to designated local directories without external connectors.

This design ensures that sensitive operations, particularly writing to external systems or generating final artifacts, are performed by components that have not directly processed untrusted inputs.

Cross-agent communication is managed through [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) mechanisms. Agents do not directly call each other; instead, an agent emits a [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) which is then routed by an orchestrator (e.g., [`../scripts/orchestrate.py`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L38) or an external event bus) as a new steering event to the target agent. The [`orchestrate.py`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L45) script implements hard-allowlisting and schema validation for these handoff requests, ensuring secure and controlled inter-agent communication. Further details on the common architectural patterns and the three-tier security isolation model employed across various managed agents can be found in [Architecture and Security Model for Managed Agents](https://codewiki.google/github.com/anthropics/financial-services#financial-services-managed-agent-cookbooks-architecture-and-security-model-for-managed-agents). For information on standardized deployment mechanisms, refer to [Agent Deployment and Configuration](https://codewiki.google/github.com/anthropics/financial-services#financial-services-managed-agent-cookbooks-agent-deployment-and-configuration). Specific functionalities of individual managed agent templates are detailed in [Specific Managed Agent Cookbooks](https://codewiki.google/github.com/anthropics/financial-services#financial-services-managed-agent-cookbooks-specific-managed-agent-cookbooks).

### Architecture and Security Model for Managed Agents

Managed agents in the financial services offering utilize a common architectural pattern and a three-tier security isolation model, primarily designed to handle untrusted documents and manage permissions. This architecture ensures secure processing and communication across different agents.

A central aspect of the security model is the task decomposition into specialized "leaf workers," each with restricted permissions to minimize risk. A typical agent comprises three tiers:

1. **Reader Tier**: Components like [`transcript-reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L11) in the [`earnings-reviewer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L26) or [`doc-reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L15) in the [`kyc-screener`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L32) directly interact with untrusted documents. These components are restricted to [`Read`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md?plain=1#L27) and [`Grep`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/pitch-agent/README.md?plain=1#L25) tools and have no external connectors. Their output is consistently length-capped and schema-validated JSON, ensuring that only safe, structured data is passed to subsequent tiers.
2. **Orchestrator Tier**: This tier, exemplified by [`model-updater`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L11) (for earnings review), [`rules-engine`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L15) (for KYC screening), or [`rollforward`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L17) (for month-end closing), does not directly interact with untrusted documents. It orchestrates the workflow using [`Read`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md?plain=1#L27), [`Grep`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/pitch-agent/README.md?plain=1#L25), [`Glob`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/kyc-screener/README.md?plain=1#L26), and [`Agent`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L12) tools. This tier often connects to read-only external financial data providers, such as [`FactSet`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L126), [`Daloopa`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L123), [`CapIQ`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/pitch-agent/README.md?plain=1#L21), or [`LSEG`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L42), enabling data retrieval without write access to external systems. In some cases, a [`critic`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L14) component independently verifies data against trusted sources before orchestration proceeds, as seen in the [`gl-reconciler`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L29).
3. **Write-holder Tier**: Designated components such as [`note-writer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L10) (for earnings review), [`escalator`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L15) (for KYC screening), [`pack-writer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L12) (for meeting preparation), or [`publisher`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L16) (for valuation review) are responsible for generating final outputs. These components also avoid direct interaction with untrusted documents. They are granted [`Read`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md?plain=1#L27), [`Write`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L20), and [`Edit`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/CLAUDE.md?plain=1#L31) tools but typically lack external connectors, writing solely to designated output directories (e.g., [`./out/`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/pitch-agent/agent.yaml#L8)). This ensures that only one worker has [`Write`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L20) permissions at a time for sensitive outputs, maintaining artifact isolation. The [`model-builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L27) further refines this by assigning sandboxed [`Bash`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/pitch-agent/README.md?plain=1#L26) access to its [`builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L27) component, allowing for controlled script execution during model generation, with an [`auditor`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L31) component subsequently re-checking results.

Inter-agent communication is managed through a [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) mechanism. Instead of direct calls between agents, an agent emits a [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) that contains information for routing. An orchestrator, such as the [`scripts/orchestrate.py`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) script, intercepts these requests, validates them against predefined schemas and allowlisted targets, and then routes them as new steering events to the appropriate target agent. This pattern, detailed in [`managed-agent-cookbooks/README.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md), supports one level of delegation, meaning an orchestrator can call workers, but workers cannot call further sub-agents, enhancing control and predictability in complex financial workflows. For example, the [`earnings-reviewer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L26) can issue a [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) to the [`model-builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L27) if an earnings-driven thesis change necessitates rebuilding a financial model. Similarly, the [`valuation-reviewer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L28) can hand off flagged portfolio companies to the [`gl-reconciler`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L29) for further investigation.

These manifests, such as those found in [`managed-agent-cookbooks`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks), resolve system prompts (which can be inlined from a [`file`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/LICENSE#L107) or specified as [`text`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/check.py#L66)), [`skills`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/examples/python-bootstrap/config.py#L25) (which can reference [`from_plugin`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/check.py#L99) to upload all skills from a plugin directory or a specific [`path`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/CLAUDE.md?plain=1#L31) to a skill), and [`callable_agents`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L87) (which reference other [`agent.yaml`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L34) files) into the appropriate fields for the [`POST /v1/agents`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) API call. The [`callable_agents`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L87) feature currently supports only one level of delegation, meaning an orchestrator can call workers, but workers cannot call further subagents. Each agent also typically includes a [`steering-examples.json`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/CLAUDE.md?plain=1#L25) file to demonstrate how to initiate and guide the agent's workflow. More details on the architecture and security model can be found in [Architecture and Security Model for Managed Agents](https://codewiki.google/github.com/anthropics/financial-services#financial-services-managed-agent-cookbooks-architecture-and-security-model-for-managed-agents).

This agent employs a three-tier security model with a [`transcript-reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L11) for untrusted documents, a [`model-updater`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L11) orchestrator, and a [`note-writer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L10) that produces an updated model and notes. It can also initiate a [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) for the [`model-builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L27) to rebuild a Discounted Cash Flow (DCF) model when an earnings-driven thesis change occurs.

For financial reconciliation tasks, the [`gl-reconciler`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L29) managed agent, defined in [`managed-agent-cookbooks/gl-reconciler`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/gl-reconciler), identifies discrepancies between general ledger and subledger data for specific trade dates and asset classes. It determines root causes and generates exception reports. Its architecture includes a [`reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/validate.py#L8) for untrusted statements, an [`Orchestrator`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/kyc-screener/README.md?plain=1#L26) that manages the reconciliation process using read-only GL and subledger Micro-Core Processors (MCPs), and a [`resolver`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L14) that creates the exception report. A [`critic`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L14) component independently verifies breaks before they are handed off to the [`resolver`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L14). This agent can hand off verified breaks to the [`month-end-closer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L30) agent.

The [`kyc-screener`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L32) managed agent, configured in [`managed-agent-cookbooks/kyc-screener`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/kyc-screener), processes onboarding documents, runs a rules engine, and screens for sanctions and Politically Exposed Persons (PEP). It flags missing information and recommends a risk rating. Its three-tier isolation architecture includes a [`doc-reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L15) for untrusted documents, a [`rules-engine`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L15) orchestrator that connects to a read-only screening service, and an [`escalator`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L15) that outputs an Excel file for further action.

Market research is handled by the [`market-researcher`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L25) managed agent, detailed in [`managed-agent-cookbooks/market-researcher`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/market-researcher). This agent follows a structured workflow from sector analysis to an industry overview, competitive landscape, peer comparisons, ideas shortlist, and finally a research note. It uses a [`sector-reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L10) for untrusted documents, a [`comps-spreader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L10) orchestrator connecting to Capital IQ and FactSet, and a [`note-writer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L10) that produces a research primer. This agent can hand off an ideas shortlist as a [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) to a [`model-builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L27) agent.

The [`model-builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L27) managed agent, defined in [`managed-agent-cookbooks/model-builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/model-builder), constructs various financial models, including DCF, Leveraged Buyout (LBO), 3-statement, and comparable company analyses, outputting them as an Excel file. Its security model isolates artifacts and re-verifies them through a [`data-puller`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L13) that sources inputs from Capital IQ and Daloopa, a [`builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L27) that generates the Excel model, and an [`auditor`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L31) that re-checks ties and balances. The [`builder`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L27) is the sole component with write permissions and operates within a sandboxed Bash environment.

For preparing briefing packs for client meetings, the [`meeting-prep-agent`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L24), described in [`managed-agent-cookbooks/meeting-prep-agent`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/meeting-prep-agent), integrates with external workflow engines. It uses a [`profiler`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L12) that connects to CRM and Capital IQ, a [`news-reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L12) for untrusted documents, and a [`pack-writer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L12) that generates the briefing pack. The [`pack-writer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L12) never directly accesses untrusted content.

The [`month-end-closer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L30) managed agent, outlined in [`managed-agent-cookbooks/month-end-closer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/month-end-closer), processes untrusted documents like invoices and vendor statements to facilitate month-end closing procedures. It uses a [`ledger-reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L17) to interact with untrusted documents, a [`rollforward`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L17) orchestrator that connects to an internal general ledger, and a [`poster`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L17) that generates a close package and stages Journal Entry drafts without direct posting. This agent processes [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) events from the [`gl-reconciler`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L29) to incorporate verified breaks into close commentary.

Pre-generated LP statements are audited by the [`statement-auditor`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L31) managed agent, specified in [`managed-agent-cookbooks/statement-auditor`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/statement-auditor). It employs a [`statement-reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L18) for untrusted statements, a [`reconciler`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L12) orchestrator that connects to NAV data, and a [`flagger`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L18) that produces a sign-off sheet recommending "pass/hold" actions. Final distribution still requires human approval.

Finally, the [`valuation-reviewer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L28) managed agent, detailed in [`managed-agent-cookbooks/valuation-reviewer`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/valuation-reviewer), ingests GP valuation packages, runs valuation templates, and stages LP reporting. Its three-tier isolation architecture includes a [`package-reader`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L16) for untrusted GP documents, a [`valuation-runner`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L16) orchestrator connecting to portfolio data, and a [`publisher`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/managed-agent-cookbooks/README.md?plain=1#L16) that generates an LP pack. This agent can hand off flagged portfolio companies to the [`gl-reconciler`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L29) via a [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85).

These managed agents are typically deployed using a script like [`deploy-managed-agent.sh`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L45) and configured via [`agent.yaml`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L34) files, resolving to [`POST /v1/agents`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) API fields. Their interactions and workflows can be orchestrated using mechanisms like [`handoff_request`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L85) that are routed by a central orchestrator. For deployment and configuration details, refer to [Agent Deployment and Configuration](https://codewiki.google/github.com/anthropics/financial-services#financial-services-managed-agent-cookbooks-agent-deployment-and-configuration). The common architectural patterns and security model for these agents are further discussed in [Architecture and Security Model for Managed Agents](https://codewiki.google/github.com/anthropics/financial-services#financial-services-managed-agent-cookbooks-architecture-and-security-model-for-managed-agents).

## Claude for Microsoft 365 Integration

This section describes the integration of Claude's Office add-in with custom large language model (LLM) backends for Microsoft 365. It outlines the administrative tools for provisioning cloud resources, generating manifests, and configuring users within the Microsoft 365 environment. The add-in authenticates users through Entra ID tokens, applies Role-Based Access Control (RBAC) rules, and dynamically delivers configuration, ensuring a seamless experience within the Microsoft 365 ecosystem.

The core of this integration is the ability to configure the Claude Office add-in to utilize various custom LLM backends, such as Google Vertex AI, AWS Bedrock, Azure AI Foundry, or a generic LLM gateway, instead of relying solely on Anthropic's API. This flexibility allows organizations to leverage their existing cloud infrastructure and LLM services. The setup process is guided by an interactive wizard found in [`claude-for-msft-365-install`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install), which assists administrators in choosing an API backend and configuring the necessary cloud resources.

Administrators use a [`build-manifest.mjs`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L271) script to generate a customized manifest XML file, as detailed in [`claude-for-msft-365-install/commands/manifest.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/manifest.md). This manifest embeds configuration keys as URL query parameters, which the add-in reads upon startup. These parameters dictate how the add-in connects to the chosen backend, handles authentication, and manages telemetry.

User-specific configurations are managed through two primary mechanisms. The first involves Azure AD extension attributes, described in [`claude-for-msft-365-install/commands/update-user-attrs.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/update-user-attrs.md), where individual or bulk settings like API tokens, gateway URLs, or cloud project IDs can be stored. The second, more dynamic approach, uses a bootstrap endpoint, as outlined in [`claude-for-msft-365-install/commands/bootstrap.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md). This HTTPS GET handler delivers dynamic JSON configuration at runtime, overriding manifest and extension attribute settings. This endpoint is crucial for delivering structured data such as lists of authorized skills and MCP servers, and managing configuration expiry. A reference implementation of this bootstrap endpoint, built with Python and FastAPI, is provided in [`claude-for-msft-365-install/examples/python-bootstrap`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/examples/python-bootstrap), which also handles Entra ID token validation and RBAC rule application.

Authentication within the Microsoft 365 environment leverages Entra ID (formerly Azure AD). When [`entra_sso=1`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L47) is configured in the manifest, the add-in acquires an Entra ID token, which is then used for authentication with backend services or the bootstrap endpoint. The system incorporates RBAC rules to define which skills and MCP servers a user can access, based on their Entra ID claims (e.g., user ID, group memberships), ensuring controlled and secure access to functionalities. Administrative consent, detailed in [`claude-for-msft-365-install/commands/consent.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/consent.md), is a one-time tenant-wide approval process necessary for the add-in's application registration and access to Microsoft Graph API permissions (e.g., [`Mail.ReadWrite`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/consent.md?plain=1#L49) for Outlook integration).

The deployment process involves uploading the generated [`manifest.xml`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L9) to the M365 Admin Center. Before deployment, administrators can verify model reachability and functionality using [`curl`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/deploy-managed-agent.sh#L27) commands. Troubleshooting and debugging guidance is available in [`claude-for-msft-365-install/commands/debug.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md) to address common issues such as stale configurations, add-in visibility, and sign-in failures, offering solutions like clearing caches and using browser developer tools.

### Administrative Tooling and Deployment

The integration of Claude within Microsoft 365 environments is facilitated through an Office add-in, which can be configured to use custom large language model backends. The administrative tooling within the [`claude-for-msft-365-install`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install) directory provides guided workflows for enterprise administrators to provision necessary cloud resources, generate and manage add-in manifests, handle administrative consent, and oversee the overall deployment process.

The initial setup begins with an interactive wizard provided by the [`/claude-for-msft-365-install:setup`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L145) command, which guides administrators through choosing an API backend (such as [`gateway`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L182), Google [`vertex`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L69), AWS [`bedrock`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L103), or Azure AI [`foundry`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L215)). This wizard then helps configure specific cloud resources, including enabling necessary API services, creating IAM OIDC providers, or obtaining API keys. It also addresses deployment specifics for different Office applications (e.g., Excel, Word, PowerPoint vs. Outlook) and helps determine whether configuration parameters will be managed org-wide or on a per-user basis.

A central component of the deployment process is the generation of a customized [`manifest.xml`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L9) file, which is handled by the [`/claude-for-msft-365-install:manifest`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/README.md?plain=1#L20) command. This manifest is responsible for embedding configuration keys as URL query parameters, enabling the add-in to communicate with the chosen backend. The manifest generation process supports various cloud providers and allows for the configuration of Microsoft Graph access, Entra SSO, telemetry settings, and optional inference headers for cost allocation. For detailed information on manifest generation, refer to [Manifest Generation and Configuration](https://codewiki.google/github.com/anthropics/financial-services#claude-for-microsoft-365-integration-administrative-tooling-and-deployment-manifest-generation-and-configuration).

Administrative consent is a critical step, especially when [`entra_sso=1`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L47) is enabled or when deploying the Outlook manifest. The [`/claude-for-msft-365-install:consent`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/README.md?plain=1#L21) command provides the necessary Azure admin consent URLs to grant the application required permissions, such as [`Mail.ReadWrite`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/consent.md?plain=1#L49) for Outlook. This ensures the add-in can interact with Microsoft services as needed.

After generating the manifest, administrators can verify the reachability of the configured Claude model using provided [`curl`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/deploy-managed-agent.sh#L27) commands before deploying the add-in. The deployment itself typically involves uploading the generated [`manifest.xml`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L9) to the M365 Admin Center and assigning it to users. Troubleshooting and debugging guidance is also available within the [`claude-for-msft-365-install/commands/debug.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md) document to help diagnose and resolve common issues encountered during deployment and operation, such as stale configurations or sign-in failures. For more comprehensive troubleshooting, see [Troubleshooting and Debugging](https://codewiki.google/github.com/anthropics/financial-services#claude-for-microsoft-365-integration-troubleshooting-and-debugging).

Per-user configuration is a significant aspect, allowing for granular control over settings beyond what the static manifest can provide. This can be achieved either through Azure AD extension attributes, managed via the [`/claude-for-msft-365-install:update-user-attrs`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/README.md?plain=1#L22) command, or dynamically through a bootstrap endpoint, as described in [Dynamic Configuration with Bootstrap Endpoint](https://codewiki.google/github.com/anthropics/financial-services#claude-for-microsoft-365-integration-dynamic-configuration-with-bootstrap-endpoint). This layered approach ensures flexibility, allowing administrators to define broad organizational defaults while also supporting highly specific user overrides.

#### Manifest Generation and Configuration

The process of integrating the Claude add-in with Microsoft 365 applications involves generating a customized Office add-in manifest XML file. This manifest acts as the primary configuration vehicle, allowing an administrator to specify how the add-in connects to various cloud providers and utilizes authentication mechanisms.

The manifest generation process typically begins by fetching a base template, which can be either for general Office applications (Excel, Word, PowerPoint) or specifically for Outlook. Configuration parameters are then appended to this base template as URL query parameters. These parameters are subsequently read by the add-in at startup to determine its operational behavior.

The manifest supports various cloud providers for the backend Large Language Model (LLM) services. For example, it can be configured for Google's Vertex AI by specifying [`gcp_project_id`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L42), [`gcp_region`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L42), and OAuth client credentials. For AWS Bedrock, it requires an [`aws_role_arn`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L43) and [`aws_region`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L43). Azure AI Foundry integration uses [`azure_resource_name`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L44) and [`azure_api_key`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L44). A generic [`Gateway`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L182) option is also available, requiring a [`gateway_url`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L41) and [`gateway_token`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L188), with the flexibility to specify the [`gateway_api_format`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L41).

For Outlook-specific deployments, the manifest enables configuration for Microsoft Graph access, which often necessitates a one-time tenant-wide administrative consent. Organizations with strict security policies can opt to use a custom single-tenant Entra application's client ID for Microsoft Graph permissions by setting [`graph_client_id`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L62).

Authentication within the add-in can be managed through Entra Single Sign-On (SSO) by enabling [`entra_sso=1`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L47). This allows the add-in to acquire an Entra ID token, which is crucial for authenticating with various backend services and dynamic configuration endpoints. If a custom [`graph_client_id`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L62) is specified for Entra SSO, the associated Entra application must be registered as a single-tenant Single-page application.

Telemetry settings can also be defined within the manifest, allowing OpenTelemetry (OTLP) traces to be routed to a custom collector via [`otlp_endpoint`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/manifest.md?plain=1#L118), with optional [`otlp_headers`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/manifest.md?plain=1#L123) for authentication and [`otlp_resource_attributes`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/manifest.md?plain=1#L127) for adding metadata to traces. Additionally, [`inference_headers`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L188) can be specified as a JSON object to attach extra HTTP headers to requests sent to the [`gateway_url`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L41), which is useful for cost allocation or accounting purposes.

Beyond static manifest configurations, more dynamic, per-user settings can be provided through a [Dynamic Configuration with Bootstrap Endpoint](https://codewiki.google/github.com/anthropics/financial-services#claude-for-microsoft-365-integration-dynamic-configuration-with-bootstrap-endpoint). This [`bootstrap_url`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/manifest.md?plain=1#L92) specifies an HTTPS endpoint from which the add-in fetches JSON containing per-user configurations, including provider keys, MCP servers, and skills. These dynamic settings override any corresponding values defined directly in the manifest or via Azure AD extension attributes.

The generation process typically utilizes a script, such as [`build-manifest.mjs`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L271) found in [`claude-for-msft-365-install/commands`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands), to construct the manifest XML file. This script takes key-value pairs as arguments, which correspond to the various configuration parameters. After generation, the manifest can be validated using tools like [`office-addin-manifest validate`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L275) to ensure it conforms to the required schema. Troubleshooting guidance, detailed in [`claude-for-msft-365-install/commands/debug.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md), helps address common issues during deployment, such as manifest updates not taking effect due to caching or specific host removals. See [Agent Deployment and Configuration](https://codewiki.google/github.com/anthropics/financial-services#financial-services-managed-agent-cookbooks-agent-deployment-and-configuration) for additional information on deployment.

User-specific settings for the Claude for Microsoft 365 add-in can be managed and delivered using Azure AD extension attributes. These attributes are then read by the add-in from the user's ID token. Each attribute value has a maximum length of 256 characters.

Individual user updates can be performed using [`az rest`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L215) with a [`PATCH`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L292) method to modify a specific extension attribute for a user. For bulk updates, scripts like [`apply.sh`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/update-user-attrs.md?plain=1#L69) (for macOS/Linux) and [`apply.ps1`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/update-user-attrs.md?plain=1#L90) (for Windows) are provided. These scripts securely read user data and attribute values from a CSV file and apply them via [`az rest`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L215), which is particularly useful for managing sensitive information like [`gateway_token`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L188) without exposing values in command history. Verification of attribute values can be done using [`GET`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L7) requests with [`az rest`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L215).

Common issues during this process include [`404`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L301) errors for incorrect user principal names (UPNs) or [`403`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L304) errors due to insufficient [`User.ReadWrite.All`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/update-user-attrs.md?plain=1#L106) permissions. Propagation delays can also occur due to Security Token Service (STS) token caching, requiring users to wait or restart their Office applications. For more details on the setup process, refer to [Administrative Tooling and Deployment](https://codewiki.google/github.com/anthropics/financial-services#claude-for-microsoft-365-integration-administrative-tooling-and-deployment), and specifically for manifest configuration, see [Manifest Generation and Configuration](https://codewiki.google/github.com/anthropics/financial-services#claude-for-microsoft-365-integration-administrative-tooling-and-deployment-manifest-generation-and-configuration).

### Dynamic Configuration with Bootstrap Endpoint

The dynamic bootstrap endpoint provides a mechanism for the Claude Office add-in to retrieve per-user configuration settings from a custom server. This endpoint, typically an HTTPS GET handler, delivers configuration as a JSON object, allowing for granular control over add-in behavior that goes beyond static manifest settings or Azure AD extension attributes. It functions as part of a tiered configuration system where values from the bootstrap endpoint override those defined in the manifest or user extension attributes.

A key aspect of this endpoint is its ability to deliver structured data, such as definitions for Microsoft Cloud Policy (MCP) servers and skills, which cannot be adequately represented by simple string attributes. The endpoint's response can include various configuration fields, such as cloud-specific provider keys (e.g., [`gateway_url`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L41), [`aws_role_arn`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L43)), OpenTelemetry (OTLP) telemetry settings, [`inference_headers`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L188) for cost allocation, and [`disabled_features`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L252). To ensure efficiency, it is recommended that the endpoint only returns fields that differ from default configurations.

Security is paramount for the bootstrap endpoint. When Entra SSO is enabled (via [`entra_sso=1`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L47) in the manifest), the endpoint validates the JSON Web Token (JWT) present in the [`Authorization`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L203) header of incoming requests. This validation process includes checking the [`aud`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L114) (audience), [`iss`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/manifest.md?plain=1#L81) (issuer), [`exp`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L138) (expiration), and [`oid`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L139) (object ID) claims, along with verifying the token's signature using Microsoft's JSON Web Key Set (JWKS). Additionally, the endpoint must adhere to Cross-Origin Resource Sharing (CORS) policies, specifically requiring [`Access-Control-Allow-Origin: https://pivot.claude.ai`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L67) to prevent browser security restrictions, especially for any URLs specified within the bootstrap response itself (e.g., [`mcp_servers[].url`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L66), [`skills[].url`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L66)).

The endpoint also supports template interpolation, allowing string values within the configuration to include placeholders (e.g., [`{{key}}`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L42)) that are resolved against the merged configuration chain. This enables dynamic construction of URLs or other values based on preceding configuration layers. To manage caching, the response can include a [`bootstrap_expires_at`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/bootstrap.md?plain=1#L262) timestamp, informing the add-in when its current configuration becomes stale and should be re-fetched.

A reference implementation for such a bootstrap endpoint is provided in [`claude-for-msft-365-install/examples/python-bootstrap`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/examples/python-bootstrap). This Python-based FastAPI application demonstrates how to validate Entra ID tokens, apply Role-Based Access Control (RBAC) rules based on user claims (like object ID and group memberships), and dynamically resolve the [`skills`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/examples/python-bootstrap/config.py#L25) and [`mcp_servers`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/examples/python-bootstrap/config.py#L40) accessible to the user. The [`resolve`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/examples/python-bootstrap/app.py#L35) function within this implementation acts as a rule engine, applying the first matching rule from a predefined set to determine the user's authorized configuration. The configuration for this example, including authentication parameters, available skills, MCP servers, and RBAC rules, is externalized to [`claude-for-msft-365-install/examples/python-bootstrap/config.py`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/examples/python-bootstrap/config.py) for ease of customization. ### Troubleshooting and Debugging

Guidance for diagnosing and resolving common issues encountered during the deployment and operation of the Claude for Microsoft 365 add-in is provided in [`claude-for-msft-365-install/commands/debug.md`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md). This documentation facilitates triage based on symptoms and offers solutions for stale configurations, connection failures, add-in visibility problems, and sign-in issues. It also includes instructions for accessing browser developer tools for deeper debugging.

One of the initial steps in troubleshooting is to analyze the "Error Paste" details from the add-in. This output provides crucial information such as [`Request:`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md?plain=1#L37), [`Manifest params:`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md?plain=1#L41), and [`Raw error:`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md?plain=1#L45), which can help identify discrepancies between the expected and actual configuration, version mismatches, or specific API/network failures.

Issues related to stale configuration, where manifest updates are not reflected, are often resolved by clearing both [`Service`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md?plain=1#L75) and [`Client`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L111) layer caches. This involves specific commands for macOS ([`rm -rf`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/deploy-managed-agent.sh#L55)) and Windows ([`rd /s /q`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md?plain=1#L102)) to force a client-side refresh. The document also addresses scenarios where the add-in is not visible within Office applications, which might stem from incorrect user assignments in the M365 Admin Center or misconfigurations in the [`Hosts`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md?plain=1#L124) list within [`manifest.xml`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/setup.md?plain=1#L9).

For authentication-related problems, especially [`sign-in popup`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/claude-for-msft-365-install/commands/debug.md?plain=1#L19) failures or loops, the documentation directs administrators to guidance on generating a [`consent URL`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L54). More information on this can be found in [Administrative Tooling and Deployment](https://codewiki.google/github.com/anthropics/financial-services#claude-for-microsoft-365-integration-administrative-tooling-and-deployment). Furthermore, detailed instructions are provided for attaching browser developer tools to the embedded WebView of the add-in. This enables in-depth debugging of JavaScript errors, network issues, and logs, with specific steps outlined for both macOS (Safari Web Inspector) and Windows (Edge DevTools).

### Fund Administration Vertical Plugin

The Fund Administration plugin provides a suite of specialized financial accounting skills designed to automate and streamline critical processes in fund administration. These skills are valuable for month-end close, audit support, and management reporting by providing detailed analysis and reports.

Key functionalities include:

- **Period-End Accrual Schedules**: The [`accrual-schedule`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/month-end-closer/agents/month-end-closer.md?plain=1#L32) skill, located in [`plugins/vertical-plugins/fund-admin/skills/accrual-schedule`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/accrual-schedule), generates a table detailing accruals for a given entity, period, and accrual policy list. For each accrual, it calculates the "This-period accrual" by subtracting already booked amounts from the period's portion of the full amount. It also drafts journal entries for controller approval, explicitly marking them "Do not post." The skill uses a reader worker to process [`untrusted`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/orchestrate.py#L9) invoices and vendor statements.
- **GL-to-Subledger Reconciliations**: The [`gl-recon`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/gl-reconciler/agents/gl-reconciler.md?plain=1#L33) skill, found in [`plugins/vertical-plugins/fund-admin/skills/gl-recon`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/gl-recon), performs reconciliations between general ledger extracts and subledger extracts. It normalizes data, matches records, and categorizes discrepancies into buckets like "Amount break" or "Timing break." The skill then classifies the likely cause of each break (e.g., [`Timing`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/gl-reconciler/skills/gl-recon/SKILL.md?plain=1#L29), [`FX`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L114), [`Mapping`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/pitch-agent/skills/pitch-deck/SKILL.md?plain=1#L70)) and generates a detailed break report and a summary.
- **Reconciliation Break Root-Cause Analysis**: The [`break-trace`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/gl-reconciler/agents/gl-reconciler.md?plain=1#L33) skill, described in [`plugins/vertical-plugins/fund-admin/skills/break-trace`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/break-trace), diagnoses reconciliation breaks identified by [`gl-recon`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/gl-reconciler/agents/gl-reconciler.md?plain=1#L33). It pulls details from both internal GL and subledger systems, compares attributes, and identifies the root cause (e.g., [`timing break`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/pitch-agent/skills/dcf-model/SKILL.md?plain=1#L305), [`FX break`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L112), [`mapping break`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/pitch-agent/skills/pitch-deck/SKILL.md?plain=1#L70)). The output includes the root cause, an owner for resolution, and a suggested action. This skill is for diagnosis only and does not perform adjustments.
- **LP Statement Validation**: The [`nav-tieout`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/nav-tieout/SKILL.md?plain=1#L2) skill, located in [`plugins/vertical-plugins/fund-admin/skills/nav-tieout`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/nav-tieout), validates Limited Partner (LP) statements against Net Asset Value (NAV) packs. It independently recalculates an LP's capital account using data from the NAV pack and performs a line-by-line comparison with the statement, flagging discrepancies. It also checks for alignment of ending capital, fund NAV, and commitment figures against the commitment register.
- **Balance Sheet Roll-Forward Schedules**: The [`roll-forward`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/roll-forward/SKILL.md?plain=1#L6) skill, found in [`plugins/vertical-plugins/fund-admin/skills/roll-forward`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/roll-forward), constructs a roll-forward schedule that ties a beginning balance to an ending balance for specified accounts, entities, and periods. It accounts for various activities such as [`Additions / new activity`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/roll-forward/SKILL.md?plain=1#L3), [`Accruals booked this period`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/roll-forward/SKILL.md?plain=1#L8), and [`Reclasses / adjustments`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/roll-forward/SKILL.md?plain=1#L14), requiring GL queries for each activity line. The schedule [`must foot`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/roll-forward/SKILL.md?plain=1#L29), with any discrepancies surfaced as unexplained items.
- **Financial Variance Commentary**: The [`variance-commentary`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins/month-end-closer/agents/month-end-closer.md?plain=1#L32) skill, detailed in [`plugins/vertical-plugins/fund-admin/skills/variance-commentary`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins/fund-admin/skills/variance-commentary), generates commentary for P&L and balance sheet items. It compares current-period actuals, prior-period actuals, and budget data, flagging lines for commentary based on materiality thresholds or an "always comment" list. For flagged lines, it provides a one-sentence explanation of the movement, sourcing drivers from underlying GL activity, and outputs a commentary table and a short narrative summary.

The [`scripts`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L45) directory within the financial services repository contains utilities designed to automate the deployment, validation, and management of AI agents. These scripts facilitate operational aspects such as checking configurations, deploying agents to the Anthropic API, orchestrating interactions between different agents, and ensuring skill consistency across the platform.

A core function of these scripts is to validate various configuration files and manifests. The [`scripts/check.py`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/check.py) script ensures consistency and correctness across plugin and managed agent cookbooks. This includes parsing YAML and JSON files, validating agent markdown frontmatter, and resolving cross-file references. It also checks for the correct bundling of skills and validates marketplace source paths. For the deployment of managed agents, the [`scripts/deploy-managed-agent.sh`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/deploy-managed-agent.sh) script automates the process of submitting agent configurations to the Anthropic API. This script handles the resolution of manifest conveniences, such as inlining system prompts from files, uploading skills, and recursively deploying sub-agents, ultimately preparing a fully resolved JSON payload for the API. The integrity of these deployments is further verified by [`scripts/test-cookbooks.sh`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/test-cookbooks.sh), which performs dry runs of deployments and validates the generated agent configurations for proper structure, non-empty system prompts, and the absence of unwanted schemas.

Beyond deployment, the scripts also manage inter-agent communication and skill synchronization. The [`scripts/orchestrate.py`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/orchestrate.py) script provides a reference for managing handoffs between different AI agents. It processes incoming messages, detects handoff requests embedded in the agent's output, validates these requests against predefined schemas and allowed targets, and then directs the conversation to the appropriate target agent. This orchestration mechanism is crucial for enabling complex workflows that span multiple specialized agents. To maintain consistency across the different agents, the [`scripts/sync-agent-skills.py`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/sync-agent-skills.py) script synchronizes bundled agent skills. It ensures that agents leverage the latest versions of canonical skill definitions by propagating updates from a central source to all agent-specific skill directories. Finally, the [`scripts/validate.py`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/validate.py) script offers a mechanism for harness-side schema validation, ensuring that the structured output from sub-agents adheres to expected JSON schemas before further processing by orchestrators.

### Managed Agent Deployment and Validation

The deployment and validation process for managed agents within the financial services offering is automated through a series of scripts designed to streamline configuration, deployment to the Anthropic API, and verification of the generated agent bodies. This automation supports the lifecycle of an agent from definition to operational readiness.

Deployment is initiated by providing an agent's YAML manifest, which serves as a declarative blueprint for the agent's capabilities. This manifest allows for certain conveniences that are resolved during deployment. For instance, system prompts, which guide an agent's behavior, can be defined as separate files and are inlined into the main agent configuration during the deployment process. Similarly, specialized skills required by an agent are uploaded to the Anthropic API, and their identifiers are then incorporated into the agent's payload. This approach ensures that skill definitions are managed centrally and linked dynamically at deployment. Furthermore, if an agent is composed of multiple sub-agents, the deployment mechanism recursively processes and deploys these sub-agents first, subsequently integrating them into the parent agent's configuration. The [`scripts/deploy-managed-agent.sh`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/deploy-managed-agent.sh) script orchestrates these steps, including environment variable substitution within YAML manifests to allow for dynamic configuration based on the deployment environment.

Once an agent's configuration is prepared, a validation process ensures its correctness and adherence to established patterns. This involves checking the generated JSON body for well-formedness, ensuring that system prompts are not empty, and verifying that the agent's structure maintains a specific depth, particularly for sub-agents. The validation also confirms that certain undesirable configurations, such as [`output_schema`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/validate.py#L9) fields, are not present in the final agent configuration that is sent to the API. This validation is performed by the [`scripts/test-cookbooks.sh`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/test-cookbooks.sh) script, which simulates deployments in a dry-run mode and applies a series of checks on the resulting agent configurations. This two-stage process of automated deployment and rigorous validation helps maintain the integrity and functionality of the managed agents within the financial services ecosystem. Further details on the architecture and security considerations for these agents can be found in the [Architecture and Security Model for Managed Agents](https://codewiki.google/github.com/anthropics/financial-services#financial-services-managed-agent-cookbooks-architecture-and-security-model-for-managed-agents) section.

### Agent Handoff Orchestration

Agent handoff orchestration manages the seamless transition of conversations between different AI agents. This process involves identifying when an agent intends to transfer a user's request, extracting the necessary information for the handoff, and then directing the conversation to the appropriate target agent.

The core mechanism for handoff orchestration involves processing an agent's output to detect specific handoff requests. These requests are typically embedded within the agent's text output as structured data, often in JSON format. A regular expression is used to identify these embedded requests. Once detected, the handoff request is parsed to extract key information, including the intended target agent and a payload containing the event and context relevant to the handoff.

A critical aspect of this system is the validation of handoff requests. This involves two main checks: first, validating the target agent against a predefined allowlist to ensure that conversations can only be steered to authorized agents. This measure prevents unauthorized or unexpected agent interactions. Second, the payload of the handoff request is validated against a JSON schema to ensure its structure and content conform to expected formats, helping to mitigate potential injection risks and ensure data integrity.

Upon successful validation, the system initiates a "steer" action, effectively transferring the conversation to the designated target agent with the extracted event and context. This allows for complex workflows where different specialized agents can collaborate to fulfill a user's request, each handling the part of the conversation for which it is best suited. For a more detailed understanding of the scripts involved in this process, refer to [Agent Deployment and Orchestration Scripts](https://codewiki.google/github.com/anthropics/financial-services#agent-deployment-and-orchestration-scripts).

### Skill Synchronization and Management

Agent skills can be designed to be shared and reused across multiple agent implementations. To ensure consistency and to propagate updates from canonical skill definitions, an automated process synchronizes these bundled agent skills. This mechanism guarantees that individual agent plugins utilize the latest versions of skills defined within the [`vertical-plugins`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/README.md?plain=1#L13) directory.

The script responsible for this synchronization, [`scripts/sync-agent-skills.py`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/scripts/sync-agent-skills.py), identifies canonical skill definitions, which are typically located within [`plugins/vertical-plugins`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/vertical-plugins). It then iterates through the skills bundled within each agent's plugin directory, found under [`plugins/agent-plugins`](https://github.com/anthropics/financial-services/blob/853f755a61f7bbb045c681327f46b354419030a1/plugins/agent-plugins). If a bundled skill has a corresponding canonical definition, the script replaces the bundled version with a fresh copy from the canonical source. This automated process is crucial for maintaining a consistent and up-to-date skill ecosystem across various financial services agents. For more information on the various plugins, refer to [Financial Services AI Agents and Plugins](https://codewiki.google/github.com/anthropics/financial-services#financial-services-ai-agents-and-plugins) and [Financial Services Vertical-Specific Plugins](https://codewiki.google/github.com/anthropics/financial-services#financial-services-vertical-specific-plugins).
