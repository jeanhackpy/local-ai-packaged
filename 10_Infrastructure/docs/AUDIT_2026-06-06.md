# Repository Audit — `local-ai-packaged` (Palanthai stack)

**Date:** 2026-06-06 · **Scope:** secrets hygiene, Docker/networking, Python & shell scripts, config governance · **Reviewer:** Senior AI Architect pass

---

## Executive Summary

The repo is clean on the basics that usually bite people — no secrets in the tree or git history, sane `.gitignore`, real reverse-proxy hardening in the base compose. The risk concentrates in one place: the `private` environment override binds 11 service ports to `0.0.0.0`, several of them (Neo4j, Redis, Qdrant, Ollama) with weak or no authentication, which is dangerous the moment this runs on a VPS with a public IP.

A handful of correctness bugs (dead `searxng` patch logic, an `UnboundLocalError` in `n8n_pipe.py`, a hardcoded path in `config-guard.sh`) and reproducibility gaps (`:latest` everywhere) round out the findings.

---

## Findings by Severity

### 🔴 HIGH

**H1 — `private` override exposes unauthenticated services to `0.0.0.0`**
`docker-compose.override.private.yml` publishes host ports for n8n (5678), Qdrant (6333/6334), **Neo4j (7474/7687)**, **Redis (6379)**, SearXNG (8081) and **Ollama (11434)**. Docker `ports:` binds to all interfaces by default and **punches through UFW/firewalld** via the Docker iptables chain. Neo4j ships a weak default credential, Redis/Qdrant/Ollama have no auth at all. On a public VPS this is remote DB access + an open LLM endpoint. The name "private" is misleading — it's the *most* exposed profile.
*Fix:* bind to loopback (`127.0.0.1:6379:6379`) or an SSH-tunnel/Tailscale interface for everything except what Caddy fronts. Only n8n/Supabase/Neo4j-browser should be public, and only via Caddy (TLS), not raw ports.

**H2 — Neo4j APOC unrestricted + strict validation disabled**
`neo4j/config/neo4j.conf` sets `dbms.security.procedures.unrestricted=apoc.*` and `server.config.strict_validation.enabled=false`. Unrestricted APOC enables filesystem/HTTP procedures (`apoc.load.*`, `apoc.export.*`). Combined with H1 (exposed 7687) and a weak default password, this is a path to host-level read/write. *Fix:* scope `unrestricted` to only the procedures you use, keep the bolt port off the public interface, and set a strong `NEO4J_AUTH`.

### 🟠 MEDIUM

**M1 — `public` override is a no-op for the AI stack (known, still unfixed)**
`.jules/sentinel.md` already documents this: `override.public.yml` only `!reset`s `kong`/`supavisor`, which live in the Supabase compose, not the AI base. The AI services are safe in `public` only because the base compose uses `expose:` (no published ports) — i.e. by accident, not by design. `config-guard.sh` then "protects" this file, creating a false sense of security. *Fix:* either delete the misleading override or make `private` the explicit opt-in and document that base = internal-only.

**M2 — Hardcoded personal Telegram chat ID as default**
`docker-compose.yml`: `TELEGRAM_CHAT_ID=${TELEGRAM_CHAT_ID:-471505542}`. A personal identifier baked into a public GitHub repo, and a silent wrong-recipient default if the env var is unset. *Fix:* remove the default; fail loudly if unset.

**M3 — `:latest` image tags throughout**
n8n, neo4j, ollama, qdrant, minio, caddy, searxng all pin `latest`. This defeats the entire "golden config / reproducible" goal of `config-guard.sh` — the config hash is locked but the running software drifts on every `pull`. *Fix:* pin digests or explicit versions; let Renovate/Dependabot bump them deliberately.

**M4 — Supabase `FUNCTIONS_VERIFY_JWT=false`**
Edge functions are unauthenticated by default. Fine for local, a hole if any function is reachable through Kong in public mode. *Fix:* default to `true`, opt out per-deployment.

### 🟡 LOW / Correctness

**L1 — `n8n_pipe.py` `UnboundLocalError`.** Line 136 `return n8n_response` is reachable when `messages` is empty, where `n8n_response` was never assigned → runtime crash instead of a clean error. Initialize it or return inside the branch.

**L2 — Dead `searxng` patch logic in `start_services.py`.** `check_and_fix_docker_compose_for_searxng()` does `content.replace("cap_drop: - ALL", ...)`, but the YAML is multi-line (`cap_drop:\n  - ALL`), so the match never fires. The function silently does nothing — and worse, *if* it ever matched it would mutate a `config-guard`-protected file, tripping the golden checksum. Remove it or fix the match.

**L3 — `config-guard.sh` hardcoded `BASE="/home/phil/local-ai-packaged"`.** Breaks on any other host/path (incl. this checkout). Derive from `$(git rev-parse --show-toplevel)` or script dir. It also auto-`git commit`s on `lock` — surprising side effect for a "guard" tool.

**L4 — Missing request timeouts.** `router_pipe.py` `call_ollama`/`call_openrouter` and `n8n_pipe`'s post have partial/no timeouts; `call_ollama` can hang a worker indefinitely. Add `timeout=` to every `requests` call.

**L5 — SearXNG `limiter: false`.** No rate limiting / bot protection. Acceptable *only* because Caddy keeps it internal (H1 would expose it on 8081). Keep it off the public interface.

**L6 — Repo hygiene.** ~8 stale `bolt-*`/`feat/*` remote branches; large blocks of commented-out OpenClaw/OpenWebUI/Flowise code in `start_services.py` and compose. No CI: `.github/` has only issue templates — no lint, no `docker compose config` validation, no secret scanning.

## Scalability & Security Notes

- **Network model is the core issue.** Adopt a single rule: *nothing is published except via Caddy*. Everything else talks over the `localai_default` bridge. This makes `private`/`public` overrides nearly unnecessary and removes the firewall-bypass class of bug entirely.
- **Secrets are env-driven and uncommitted** — good. Next step is moving `POSTGRES_PASSWORD`, `NEO4J_AUTH`, `N8N_ENCRYPTION_KEY` into Docker secrets or a vault rather than a flat `.env`, per your privacy-first guardrail.
- **Reproducibility:** pin images + add a CI job running `docker compose config` against each profile to catch override regressions like M1 automatically.

## Actionable Implementation (priority order)

1. **H1/H2** — Rewrite `override.private.yml` to bind `127.0.0.1` only; set strong `NEO4J_AUTH`; scope APOC `unrestricted`. *(highest impact, low effort)*
2. **M2** — Drop the hardcoded Telegram chat ID default.
3. **L1/L2** — Fix the `n8n_pipe` crash path; delete the dead SearXNG patch function.
4. **M3** — Pin image versions; add Renovate.
5. **L3** — De-hardcode `config-guard.sh` base path.
6. **M1/L6** — Remove the misleading public override; add a minimal CI (`compose config` + `gitleaks` + `ruff`).
