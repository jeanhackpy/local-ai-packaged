## coleam00/local-ai-packagedPowered by Gemini

The [`(root)`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/) repository provides a self-hosted, Docker Compose-based environment for AI development. This environment integrates various services to support the creation and deployment of AI workflows, prioritizing local execution and privacy.

Docker Compose defines and links all services within the environment. The [`start_services.py`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L143) script orchestrates the environment, managing tasks like configuration and sequential startup of Docker containers. This script also handles hardware-specific configurations and deployment environments. See [Orchestrating the AI Environment](https://codewiki.google/github.com/coleam00/local-ai-packaged#ai-development-environment-setup-orchestrating-the-ai-environment) for additional information.

The environment integrates a collection of services:

- **Workflow Automation**: [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) serves as the primary engine for building and automating AI agent workflows.
- **Local AI Models**: [`Ollama`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L5) enables running Large Language Models (LLMs) locally, with [`Open WebUI`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L5) providing a chat interface for interaction.
- **Data and Backend**: [`Supabase`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L348) offers foundational services like a Postgres database and vector store capabilities. [`Qdrant`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L55) provides an alternative vector store, and [`Neo4j`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) supports knowledge graph applications.
- **Supporting Services**: [`SearXNG`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) delivers private metasearch, [`Langfuse`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) aids in LLM observability, and [`Caddy`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) functions as a reverse proxy for exposing services.

These services communicate internally within a unified Docker network. [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) workflows are central to defining AI agent behaviors, interacting with other services and accessing local files via shared volumes. Further details on individual services and their configurations are available in [Core Components and Configuration](https://codewiki.google/github.com/coleam00/local-ai-packaged#ai-development-environment-setup-core-components-and-configuration). The [`Pipe`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L26) class in [`n8n_pipe.py`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py) facilitates interaction between external agents and [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) workflows. More information is in [N8N Workflow Integration](https://codewiki.google/github.com/coleam00/local-ai-packaged#ai-development-environment-setup-n8n-workflow-integration).

Standardized procedures for managing development tasks use GitHub issue templates. These templates facilitate clear communication for bug reports and feature requests. [GitHub Issue Management](https://codewiki.google/github.com/coleam00/local-ai-packaged#github-issue-management) provides guidelines for their use.

## AI Development Environment Setup

The [`(root)`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/) repository provides a self-hosted, Docker Compose-based environment for AI development. This environment integrates various services to support the creation and deployment of AI workflows, prioritizing local execution and privacy. Key services include [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) for automation, [`Supabase`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L348) for database and authentication, [`Ollama`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L5) for local Large Language Models (LLMs), and [`Open WebUI`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L5) for chat interaction. Additional components like [`Flowise`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) for AI agent building, [`Qdrant`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L55) and [`Neo4j`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) for specialized data storage, [`SearXNG`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) for private search, [`Langfuse`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) for observability, and [`Caddy`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) for reverse proxying are also included.

The setup process begins with cloning the repository and configuring environment variables by copying and modifying the provided [`.env.example`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L10) file. These variables define crucial settings for services, including API keys, database credentials, and hostnames. The [`start_services.py`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L143) script orchestrates the entire environment, handling tasks such as preparing the [`Supabase`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L348) configuration, generating secrets for [`SearXNG`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7), and managing the sequential startup of Docker containers. This script supports various hardware configurations (e.g., Nvidia GPU, AMD GPU, CPU-only) and deployment environments (private or public) through command-line arguments and Docker Compose profiles.

The architecture relies on [`docker-compose.yml`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L331) to define and link all services, using YAML anchors for reusability. Override files, such as those found in [`(root)`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/), manage environment-specific configurations, such as port exposures. All services operate under a unified Docker project name, [`localai`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L331), simplifying management. Internal networking allows services to communicate using their names, while [`Caddy`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) serves as a reverse proxy to expose services via local URLs. [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) workflows, often pre-built, are central to defining AI agent behaviors and can interact with other services, including accessing local files via shared volumes. For detailed insights into the orchestration logic, see [Orchestrating the AI Environment](https://codewiki.google/github.com/coleam00/local-ai-packaged#ai-development-environment-setup-orchestrating-the-ai-environment). The specific services and their configurations are further described in [Core Components and Configuration](https://codewiki.google/github.com/coleam00/local-ai-packaged#ai-development-environment-setup-core-components-and-configuration).

### Orchestrating the AI Environment

The self-hosted AI development environment is primarily orchestrated by the [`start_services.py`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L143) script located at [`start_services.py`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/start_services.py). This script automates the setup, configuration, and sequential startup of various AI services, ensuring a consistent and functional environment. It manages the entire stack, including preparing the Supabase repository, configuring [`SearXNG`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7), and launching both the Supabase and local AI services using Docker Compose. The script provides flexibility by supporting different hardware configurations and deployment environments.

Key functionalities of the orchestrator include:

- **Repository and Environment Preparation**: The script handles the initial setup for core components. This involves cloning the Supabase repository via [`clone_supabase_repo`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/start_services.py#L23) and preparing its environment files using [`prepare_supabase_env`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/start_services.py#L62). It also addresses platform-specific issues like Windows line endings with [`fix_windows_line_endings`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/start_services.py#L42).
- **[`SearXNG`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) Configuration**: For the [`SearXNG`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) service, the script generates a secret key using [`generate_searxng_secret_key`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/start_services.py#L100) and temporarily modifies the [`docker-compose.yml`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L331) file to facilitate its initial setup. This ensures [`SearXNG`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) initializes correctly before restoring security-related configurations, managed by [`check_and_fix_docker_compose_for_searxng`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/start_services.py#L172).
- **Sequential Service Startup**: To manage dependencies, the script ensures a phased startup. It first stops any existing Docker containers related to the [`localai`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L331) project with [`stop_existing_containers`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/start_services.py#L69), then initiates the Supabase services using [`start_supabase`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/start_services.py#L77), allowing them to initialize. Following this, the local AI services are launched via [`start_local_ai`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/start_services.py#L86). This sequential approach prevents common issues arising from services attempting to connect to unavailable dependencies.
- **Hardware and Deployment Flexibility**: The [`start_services.py`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L143) script supports various configurations through command-line arguments. Users can specify different Docker Compose profiles (e.g., [`cpu`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L165), [`gpu-nvidia`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L148), [`gpu-amd`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L158), [`none`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L172)) to align with their available hardware resources for services like Ollama. Additionally, it allows selecting deployment environments (e.g., [`private`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/CLAUDE.md?plain=1#L54), [`public`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L202)), which can influence network settings and security configurations. These profiles and environments are managed through Docker Compose override files, enabling dynamic adjustments without altering the base [`docker-compose.yml`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L331).

All services are managed under a unified Docker project named [`localai`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L331), simplifying visibility and control across the entire environment. Further details on the individual services and their configurations can be found in [Core Components and Configuration](https://codewiki.google/github.com/coleam00/local-ai-packaged#ai-development-environment-setup-core-components-and-configuration).

### Core Components and Configuration

The AI development environment integrates several services to create a comprehensive, self-hosted platform. These services are orchestrated using Docker Compose, configured via environment variables, and interconnected through internal and external network settings.

[`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) serves as the automation and low-code workflow engine, enabling integration between various services and the creation of AI agent workflows. Its configuration involves setting up environment variables, importing pre-built workflows, and managing sensitive node permissions through [`NODES_EXCLUDE`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L373).

[`Supabase`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L348) provides foundational services such as a Postgres database, vector store capabilities, and authentication. Its setup involves cloning the Supabase repository, preparing environment variables by copying the root [`.env`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L10) file into the appropriate directory, and ensuring its services start before other dependent AI components.

[`Ollama`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L5) facilitates running local Large Language Models (LLMs), supporting different hardware configurations including Nvidia GPUs ([`gpu-nvidia`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L148)), AMD GPUs ([`gpu-amd`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L158)), or CPU-only ([`cpu`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L165)) execution, as well as an option to exclude it entirely ([`none`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L172)) for native installations.

[`Open WebUI`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L5) offers a ChatGPT-like interface for interacting with local models and [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) agents. It integrates with [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) workflows via custom functions, such as those defined in [`n8n_pipe.py`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py), to enable conversational AI agents.

[`Flowise`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) is integrated as a no/low-code AI agent builder, allowing users to design and deploy AI agents visually.

[`Qdrant`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L55) functions as an open-source vector store, providing an alternative to Supabase for RAG applications where its performance characteristics may be preferred.

[`Neo4j`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) is included as a knowledge graph engine, supporting graph-based AI applications that require complex relationship mapping and querying. Its configuration involves specific authentication settings.

[`SearXNG`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) provides a privacy-focused metasearch engine, ensuring local and private search capabilities within the environment. Its setup includes generating a unique secret key for its settings file, `/coleam00/local-ai-packaged/searxng/settings.yml`, and temporarily adjusting Docker Compose settings for initial configuration.

[`Langfuse`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) offers an open-source LLM engineering platform for observability, providing insights into LLM application performance and behavior. Its configuration requires several environment variables for secure operation.

[`Caddy`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L7) acts as a reverse proxy, managing and exposing various services, particularly for public deployments. It handles local/private URLs and can be configured for managed HTTPS/TLS, requiring hostname and email configurations.

All services are defined and linked through [`docker-compose.yml`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L331) and environment-specific override files like [`docker-compose.override.*.yml`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/CLAUDE.md?plain=1#L45). Internal networking allows services to communicate using their service names (e.g., [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) to [`db`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L283) for Supabase Postgres). The entire environment operates under a unified Docker project name, [`localai`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L331), simplifying management. A shared volume mounted at [`/data/shared`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L420) in the [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) container facilitates file access and data exchange between services. For additional details on managing the environment, refer to [Orchestrating the AI Environment](https://codewiki.google/github.com/coleam00/local-ai-packaged#ai-development-environment-setup-orchestrating-the-ai-environment).

### N8N Workflow Integration

The environment integrates with external agents through [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) workflows, managed by the [`Pipe`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L26) class within the [`n8n_pipe.py`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py) script. This class facilitates sending user input to an [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) webhook and processing the responses received.

The [`Pipe`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L26) class is configured via a [`Valves`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L27) Pydantic model, which specifies parameters such as the [`n8n_url`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L28) for the webhook, an [`n8n_bearer_token`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L31) for authentication, and [`input_field`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L32) and [`response_field`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L33) to identify user input and N8N's output within JSON payloads.

The primary interaction method, [`pipe`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L26), sends user input, encapsulated in a JSON payload, to the configured [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) webhook. It then extracts the processed response from [`n8n`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/README.md?plain=1#L371) based on the specified [`response_field`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L33). During this process, the [`emit_status`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/n8n_pipe.py#L49) method provides status updates, including progress and error messages, back to the calling agent, with options for rate limiting and enabling/disabling status indicators. This ensures that the agent can monitor the workflow's execution and react to its outcomes.

The GitHub repository includes a dedicated template for reporting bugs, located at [`.github/ISSUE_TEMPLATE/bug_report.md`](https://github.com/coleam00/local-ai-packaged/blob/2c541913c0f97d7e2e4de3ee7e3f790fc63ce613/.github/ISSUE_TEMPLATE/bug_report.md). This template structures bug reports to ensure clarity and completeness, aiding in efficient problem diagnosis and resolution. When creating a new bug report, it automatically applies the "bug" label and sets the issue name to "Bug Report."

The template guides users to provide specific information across several key sections:

- **Description**: A general overview of the issue.
- **Steps to Reproduce**: A numbered list detailing the exact actions that lead to the bug. This section is critical for enabling maintainers to replicate the issue.
- **Expected Behavior**: A description of what was anticipated to happen.
- **Actual Behavior**: A detailed account of what occurred, contrasting it with the expected outcome.
- **Screenshots**: A section for visual evidence to support the report.
- **Environment**: Important details about the operating system (OS) and the deployment method (e.g., Docker Desktop, WSL), which can be crucial for reproducing environment-specific bugs.
- **Additional Context**: Supplementary information, such as the consistency of the issue, recent changes, or discovered workarounds.
- **Possible Solution**: An optional section for users to suggest potential fixes or root causes if they have insights.

This structured approach ensures that all necessary information is collected upfront, streamlining the bug resolution process for both reporters and maintainers.
