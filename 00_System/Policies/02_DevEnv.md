## Development Environment Standards

This document outlines the development environment standards and configurations.

## IDE Configuration Standards (see [[Development/IDE & Skills]])

1. All IDEs should use the shared skills repository
2. Extensions must be consistent across platforms
3. Version control integration is mandatory
4. Security scanning should be enabled

## Agentic Workflow Policies (see [[Development/Skills Antigravity Awesome]])

1. Skills must be version controlled
2. New skills should be documented
3. Skill updates should be tested
4. Cross-IDE compatibility is required

## IDE Configuration

- [[Development/IDE & Skills]] - Cursor as primary IDE with agentic skills
- [[Development/IDE & Skills]] - VS Code as secondary IDE
- [[Development/IDE & Skills]] - Antigravity for Google AI development
- Proper extension configuration for each IDE

## Version Control

- Git configured with proper credentials (see [[Development/IDE & Skills]])
- GitHub/GitLab accounts properly set up
- Commit signing enabled
- Branch protection rules understood

## Package Management

- Homebrew for system packages (see [[Development/IDE & Skills]])
- pipx for Python CLI tools (see [[Development/IDE & Skills#Python Setup]])
- npm/yarn for JavaScript packages
- Cargo for Rust development

## Virtual Environments

- pyenv for Python version management (see [[Development/IDE & Skills#Python Setup]])
- uv for fast virtual environments
- Docker for containerized development
- Proper isolation of projects
